> **not run** on events it should filter, and to safely handle an attacker-controlled payload. Response-only

### D1 ‚Äî Locked: the filter decision is read from the **door's returned decision**, not `AgentEvent.status`

The plan assumes an `AgentEvent` row that the test-event door does not write (R3). Three reasons to score the
returned decision rather than "fix" the door to write one:

1. **It is the same decision.** The door calls the **real** `evaluate_filters` (`playground.py:1435`) with the
   trigger's **real** `filter_conditions`, from a copy the parity gate keeps **byte-identical** to the
   gateway's (R14). The decision is real; the `agent_events` row is an *audit record of a delivery*, not the
   decision itself.
2. **Writing one would corrupt the audit log.** `agent_events` is the production event log (indexed
   `idx_agent_events_agent_received`, read by the Event Trace UI). A 20-item eval would write 20 synthetic
   "events" **indistinguishable from real deliveries**. Making them distinguishable needs a `source`
   discriminator on `agent_events` ‚Äî a separate change with its own readers. Recorded in the Gap Ledger.
3. **The purpose-built column is already there and orphaned.** `eval_run_results.matched` (R4) exists for
   exactly this and has no writer today. E-4 closes the orphan instead of manufacturing a second one.

**The honesty guard is `T-S77-009`, not the row:** the SAME payload through the **REAL event-gateway** (a real
WS-4-signed webhook, signed with the product's own `sign_webhook` ‚Äî `webhook_auth.py:242`, AST-extracted at
suite runtime exactly as `suite-76:135-153` does) must produce the SAME decision, asserted on the **real
committed `agent_events` row** the gateway writes. Static parity (T-S77-000) + live differential (T-S77-009).

### D2 ‚Äî Locked: ONE run-creation path ‚Äî `test-event` stops hand-building a second `PlaygroundRun`

R5 (no `eval_mode` ‚áí real side-effect delivery under eval) and R6 (no durable dispatch ‚áí the matched action
never runs) are **two symptoms of one design flaw** (R7). Patching each into `test_event` is the bandaid
CLAUDE.md forbids ("does this fix the class of problem, or just this instance?") and rebuilds the
`side-effecting-lost-on-declarative-runner-path.md` failure mode: a second hand-maintained builder that
silently drops a field.

**Fix the design:** extract `_create_and_dispatch_playground_run(...)` ‚Äî **1 definition, 2 call sites**
(`create_playground_run` + `test_event`), mirroring WS-4's `verify_webhook_auth`. The helper owns run
construction + Langfuse trace + durable dispatch. The **HTTP concerns stay in the endpoints** (the 403 owner
check at `playground.py:83-92` must NOT move into the helper ‚Äî `test-event` has never enforced it and moving
it would silently change that door's authz). `T-S77-000` greps 1 def / 2 call sites as a hard assertion.
